# 多人联机贪吃蛇 —— 游戏逻辑与后端接口文档

> 适用版本：v2.0（小清新绿色主题，双模式支持）
> 技术栈：Vue3 + Vite + Element Plus + Canvas 2D + Pinia + WebSocket

---

## 一、游戏完整体验流程

### 1.1 页面路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/login` | 登录页 | 用户登录/注册入口 |
| `/lobby` | 游戏大厅 | 房间列表、快速匹配、创建房间 |
| `/room/:roomId` | 游戏房间 | 玩家列表、聊天、准备/开始 |
| `/game/:roomId?mode=single\|multi` | 游戏对战 | Canvas 蛇蛇对战 |
| `/result/:gameId` | 结算页面 | 排名展示、数据统计 |

### 1.2 完整用户流程

```
登录 ──→ 游戏大厅 ──→ 创建房间/加入房间 ──→ 准备就绪 ──→ 开始游戏
                                                              │
                                                    ┌─────────┘
                                                    ▼
                                              游戏对战 (3秒倒计时)
                                                    │
                                          ┌─────────┼─────────┐
                                          ▼         ▼         ▼
                                       正常结束   玩家死亡   主动退出
                                          │         │         │
                                          ▼         ▼         ▼
                                       结算页面   观战/退出   大厅
```

## 二、游戏模式

### 2.1 单人无尽模式 (`single`)

| 属性 | 值 |
|------|-----|
| 最大玩家 | **1 人** |
| 游戏时长 | **无限**（直到玩家死亡） |
| 积分方式 | 吃食物 + 生存时间奖励 |
| 如何开始 | 创建房间即房主→直接开始；加入大厅单人房→准备→手动点开始 |
| 死亡条件 | **撞到障碍物** |

### 2.2 多人对战模式 (`multi`)

| 属性 | 值 |
|------|-----|
| 最大玩家 | **2~8 人**（创建时自选） |
| 游戏时长 | **3/5/8 分钟**（创建时自选，默认 5 分钟） |
| 积分方式 | 吃食物 + 击杀对手 + 生存奖励 + 最后存活奖励 |
| 如何开始 | 全员准备后房主点击开始；大厅加入房间满员自动开始 |
| 死亡条件 | **撞到障碍物** 或 **撞到其他蛇**（身体/头部） |

## 三、碰撞/死亡机制（全模式统一）

| 碰撞类型 | 结果 | 说明 |
|----------|------|------|
| 🧱 撞墙 | **不死亡，穿墙循环** | 从右边界出→从左边界入，同理上下 |
| 🐍 撞自己 | **不死亡，直接穿过** | 可以穿透自己的身体，不会卡住 |
| 🚧 撞障碍物 | **立即死亡** | 地图上有 5 个随机障碍物 |
| 💀 撞其他蛇头部 | **身体更长者存活**，短者死亡 | 相同时当前移动者获胜 |
| 💀 撞其他蛇身体 | **撞击者死亡**（除非有护盾） | 护盾：抵消一次撞击 |
| 🛡 护盾保护 | 护盾消失，不死亡 | 仅能抵挡一次身体撞击 |

## 四、积分/得分系统

### 4.1 积分明细

| 行为 | 得分 | 说明 |
|------|------|------|
| 🍎 吃普通食物 | **+10 分** | 地图上 90% 是普通食物 |
| 🥇 吃高分食物 | **+50 分** | 10% 概率生成，外观不同 |
| ⏱ 生存奖励 | **+20 分 / 30秒** | 每存活 30 秒自动加 |
| ⚔ 击杀对手 | **+100 分** | 撞死其他蛇（给击杀者加分） |
| 🏆 最后存活 | **+200 分** | 多人模式只剩最后一条蛇时 |
| 🧲 磁铁吸食 | 同食物分值 | 磁铁将食物吸到头位置时自动吃入计分 |

### 4.2 速度系统

| 阶段 | 速度 | 说明 |
|------|------|------|
| 初始 | **0.5 格/tick** | 起步较慢 |
| 每吃一个食物 | **+0.02** | 逐渐加速 |
| 上限 | **2.0 格/tick** | 最快速度 |
| 道具加速 (⚡) | **×1.5 倍** | 持续 5 秒 |
| 实现方式 | 步长累积器 | 小数速度也能平滑移动 |

---

## 五、道具系统

| 道具 | 图标 | 快捷键 | 效果 | 持续时间 |
|------|------|--------|------|----------|
| 加速 | ⚡ | `1` / `空格` | 移动速度 ×1.5 | 5 秒 |
| 护盾 | 🛡 | `2` | 抵挡一次撞击 | 直到被触发 |
| 磁铁 | 🧲 | `3` | 8 格范围内食物向蛇靠近 | 8 秒 |

- 道具每 **5 秒**在地图上随机补充 1 个（上限 5 个）
- 磁铁吸到蛇头位置的食物**立即自动吃入并计分**

## 六、游戏结束条件

### 6.1 单人模式

| 结束条件 | 说明 |
|----------|------|
| 🚧 玩家撞到障碍物 | 唯一死亡方式，立即结束 |

### 6.2 多人模式

| 结束条件 | 说明 |
|----------|------|
| ⏰ 游戏时间耗尽 | 倒计时归零，按积分排名 |
| 💀 只剩一条蛇存活 | 立即结束，存活者 +200 分 |
| 🚧 + 💀 所有蛇死亡 | 如有同时死亡，按积分排名 |

### 6.3 游戏结束后

1. 弹出"游戏结束"遮罩，显示排名和分数
2. 点击"查看结算"跳转到 `/result/:gameId`
3. 结算页面依次动画展示冠军→排名列表→个人数据
4. 可"返回大厅"或"再来一局"

## 七、房间逻辑

### 7.1 创建房间

- 选择模式：单人无尽 / 多人对战
- 多人模式需设置最大人数（2-8）和游戏时长（3/5/8分钟）
- 可设置房间密码
- 创建后自动成为**房主**，直接进入房间

### 7.2 加入房间

- 大厅显示房间卡片：名称、模式、人数、状态
- **已满**房间：灰色半透明，**不可点击加入**
- **游戏中**房间：不可加入
- 只有**等待中**且**未满**的房间可以加入
- 单人模式房间：0/1（等待中）→ 可加入；1/1（已满）→ 不可加入

### 7.3 准备与开始

| 场景 | 准备要求 | 开始方式 |
|------|----------|----------|
| 房主 + 单人模式 | 无需准备 | **房主直接点开始** |
| 房主 + 多人模式 | 全员准备 | **房主点开始** |
| 大厅加入 + 单人模式 | 自己准备 | **手动点开始按钮** |
| 大厅加入 + 多人模式 | 全员准备 | **满员自动开始** 或手动点开始 |

### 7.4 房主权限

- ✅ 切换其他玩家准备状态
- ❌ 踢出玩家
- ⚠ 房主离开房间**不会解散房间**（当前无此逻辑，后续可加）

## 八、数据流转

```
创建房间: LobbyView ──(sessionStorage: new_room_config)──→ RoomView
加入房间: LobbyView ──(sessionStorage: joined_room_data)──→ RoomView
开始游戏: RoomView ──(sessionStorage: game_players)──→ GameView
游戏结算: GameView ──(sessionStorage: game_result)──→ ResultView
```

所有跨页面数据通过 `sessionStorage` 传递，页面加载后立即读取并清除。

## 九、后端 API 接口

### 9.1 通信架构

```
前端 ←─── HTTP/REST ───→ 后端服务 (常规请求)
前端 ←─── WebSocket ───→ 后端服务 (游戏实时通信)
```

当前处于 **Mock 模式**（`VITE_USE_MOCK` 环境变量控制），切换真实后端时关闭 Mock 即可。

### 9.2 认证模块 `POST /api/auth`

#### `POST /api/auth/login` — 用户登录

```
请求体：
{
  "username": "admin",
  "password": "123456"
}

响应体：
{
  "user": {
    "id": "1",
    "username": "admin",
    "nickname": "管理员",
    "avatar": "",
    "level": 10,
    "score": 9999
  },
  "token": "jwt_token_string"
}
```

#### `POST /api/auth/register` — 用户注册

```
请求体：
{
  "username": "newuser",
  "password": "123456",
  "nickname": "新玩家"
}

响应体：
{
  "user": { "id": "4", "username": "newuser", ... },
  "token": "jwt_token_string"
}
```

#### `POST /api/auth/logout` — 退出登录

```
请求头：Authorization: Bearer <token>
响应体：{ "success": true }
```

#### `GET /api/auth/me` — 获取当前用户信息

```
请求头：Authorization: Bearer <token>
响应体：
{
  "id": "1", "username": "admin",
  "nickname": "管理员", "avatar": "",
  "level": 10, "score": 9999
}
```

### 9.3 房间模块 `GET/POST /api/rooms`

> 所有请求头：`Authorization: Bearer <token>`

#### `GET /api/rooms/list` — 获取房间列表

```
查询参数：
  page     int     页码（默认1）
  size     int     每页条数（默认10）
  status   string  筛选状态：all | waiting | playing | full
  keyword  string  搜索房间名关键词

响应体：
{
  "list": [
    {
      "id": "room_1",
      "name": "新手训练营",
      "hostId": "player_1",
      "hostName": "玩家1",
      "playerCount": 2,
      "maxPlayers": 4,
      "status": "waiting",       // waiting | playing | full
      "hasPassword": false,
      "gameDuration": 300,       // 秒
      "gameMode": "multi"        // single | multi
    }
  ],
  "total": 10,
  "page": 1,
  "size": 10
}
```

#### `POST /api/rooms` — 创建房间

```
请求体：
{
  "name": "欢乐派对",
  "gameMode": "multi",        // single | multi
  "maxPlayers": 6,            // 单人模式自动为1
  "gameDuration": 300,        // 单人模式为 Infinity
  "hasPassword": false,
  "password": ""

响应体：
{
  "roomId": "room_1234567890",
  "name": "欢乐派对",
  "hostId": "player_1",
  "maxPlayers": 6,
  "status": "waiting",
  "hasPassword": false,
  "gameDuration": 300,
  "gameMode": "multi"
}
```

#### `POST /api/rooms/:roomId/join` — 加入房间

```
请求体：
{ "password": "123456" }    // 可选，仅密码房需要

响应体：
{ "roomId": "room_1", "success": true }
```

#### `POST /api/rooms/:roomId/leave` — 退出房间

```
响应体：{ "success": true }
```

#### `POST /api/rooms/:roomId/start` — 房主开始游戏

```
响应体：
{ "success": true, "gameId": "game_1234567890" }
```

#### `PUT /api/rooms/:roomId` — 更新房间设置（仅房主）

```
请求体：

  "maxPlayers": 8,
```

### 9.5 WebSocket 实时通信 `ws://host/ws/game`

#### 连接建立

```
ws://<host>/ws/game?token=<jwt_token>&roomId=<roomId>
```

连接成功后，前后端通过 JSON 消息双向通信。

---

#### 客户端 → 服务端 消息

| type | data | 说明 |
|------|------|------|
| `join_room` | `{ roomId }` | 加入房间，服务端记录连接 |
| `leave_room` | `{ roomId }` | 退出房间 |
| `ready` | `{ roomId, isReady: true/false }` | 切换准备状态 |
| `start_game` | `{ roomId }` | 房主开始游戏 |
| `change_direction` | `{ direction: "up"/"down"/"left"/"right" }` | 改变蛇移动方向 |
| `speed_boost` | `{}` | 使用加速道具 |
| `use_item` | `{ itemType: "speed"/"shield"/"magnet" }` | 使用指定道具 |
| `chat_message` | `{ roomId, text }` | 发送聊天消息 |
| `ping` | `{}` | 心跳（30秒间隔） |

---

#### 服务端 → 客户端 消息

| type | data | 说明 |
|------|------|------|
| `room_update` | `{ roomId, playerList, ... }` | 房间状态更新（玩家加入/离开/准备） |
| `game_state` | 见下方帧数据结构 | 游戏帧数据（10 tick/秒广播） |
| `player_eliminated` | `{ snakeId, nickname, reason, time }` | 玩家被淘汰通知 |
| `player_kill` | `{ killerId, victimId, method }` | 击杀事件通知 |
| `game_over` | `{ gameId, duration, rankings[] }` | 游戏结束 + 结算数据 |
| `countdown` | `{ seconds }` | 游戏开始前倒计时 |
| `chat_broadcast` | `{ senderId, senderName, text }` | 聊天消息广播 |
| `pong` | `{}` | 心跳响应 |
| `error` | `{ code, message }` | 错误消息 |

### 9.6 服务端职责约定

| 职责 | 说明 |
|------|------|
| 游戏主循环 | 服务端按固定 10 tick/s 驱动所有蛇移动 |
| 碰撞检测 | 服务端判定撞墙/撞蛇/撞障碍物，决定生死 |
| 食物生成 | 服务端生成，初始 20 个，< 20 时补充 3 个/次，每 2 秒补 2 个 |
| 道具生成 | 服务端生成，初始 3 个，< 5 时每 5 秒补 1 个 |
| AI 蛇行为 | 服务端控制 AI 蛇方向（15% 概率变向，70% 趋食） |
| 积分计算 | 完全由服务端计算，前端只显示 |
| 状态广播 | 每个 Tick 广播完整帧数据给房间内所有客户端 |
| 结算数据 | 服务端生成最终排名并推送 `game_over` 事件 |

### 9.7 前端职责约定

| 职责 | 说明 |
|------|------|
| Canvas 渲染 | 接收帧数据，Canvas 2D 绘制蛇、食物、道具、障碍物 |
| 键盘输入 | 监听方向键/WASD，发送 `change_direction` |
| HUD 显示 | 显示分数、排名、倒计时、道具状态 |
| 道具快捷键 | 1/2/3 发送 `use_item`，空格发送 `speed_boost` |
| 死亡观战 | 多人模式死亡后可观战或退出 |
| 结算展示 | 排名动画、冠军展示、个人数据卡片 |
